### 2. 作業任務部署細節

#### 任務 1：搖桿圓形限位實作 (Circular Clamping Lab)
*   **目標**：打造「有物理邊界」的互動。應用勾股定理 $d = \sqrt{dx^2 + dy^2}$ 判斷距離，並利用比例關係將搖桿頭限制在圓周內，避免操作失控。
*   **Classroom 部署建議**：
    *   **極限值處理實作**：編寫一個判斷函式，當距離 $d > R$ 時，將 $dx, dy$ 等比例縮小。
    *   **除以零例外處理**：思考並寫下：如果手指剛好點在圓心 $(d=0)$，你的縮放公式是否會崩潰？該如何防護？
    *   **驗證要點**：**邊界健壯性審核**。學員是否能讓搖桿頭平滑地在圓周邊緣滑動而不會跳變？公式是否簡潔（避免了複雜的三角函數，改用比例運算優先）？

#### 任務 2：動態數據歸一化與輸出 (Data Normalization Lab)
*   **目標**：將「像素」轉化為「指令」。學會將畫布座標映射至 $[-100, 100]$ 或 $[0, 1]$ 的標準區間，並計算方向角度（利用 `Math.atan2`）。
*   **Classroom 部署建議**：
    *   **推力百分比計算**：實作 `Power = Math.min(distance / R, 1.0)`，確保輸出資料始終在 100% 以內。
    *   **象限感知角度運算**：使用 `Math.atan2(dy, dx)` 正確判斷 360 度的方向，並轉換為角度制（Degree）。
    *   **驗證要點**：**遙測準確性審核**。顯示的數值是否能隨著手指移動精準更新？當向「左後方」推時，角度與推力是否符合預期？

#### 任務 3：搖桿自動回彈與安全重置 (Auto-Centering Lab)
*   **目標**：實作「失效安全」。確保使用者放開手指時，搖桿頭能自動歸零 $(0,0)$，同時機器人接收到明確的「停止」指令。
*   **Classroom 部署建議**：
    *   **釋放事件重置**：在 `touchend` 中將全域狀態清零。
    *   **回彈動畫 (進階)**：挑戰利用 `requestAnimationFrame` 製作一個具備物理摩擦感的彈回中心效果。
    *   **驗證要點**：**操控安全性審核**。放手後，發送出的 JSON 封包中 Power 是否確實變為 0？是否有任何狀態殘留導致機器人停不下來？
